  • Container

    The Container protocol should be implemented by the instance that produces any types of the view controllers that can be considered as containers (eg: UINavigationController, UITabBarController, etc)

    The Container uses perform(embedding:) method of an ContainerAction and then populate a full stack of the view controllers that was built by the associated factories in one go. Example: Router requires to populate N-view controllers into UINavigationController‘s stack and it can do so.

    See more

    public protocol Container : AbstractFactory where Self.ViewController : ContainerViewController

  • RoutingInterceptor

    Routing interceptor is called before the actual deeplinking(navigation) happens in case it’s defined, e.g. user should be logged in.

    NB

    Interceptor is an asynchronous action. For the Router to continue routing, completion of interceptor’s execute method MUST be called. Otherwise, the Router will stay in limbo state waiting for the interceptor to finish its action.
